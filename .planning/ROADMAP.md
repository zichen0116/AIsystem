# Roadmap: 课堂预演文件上传链路

**Milestone:** v1 — 文件上传驱动的课堂预演闭环
**Phases:** 4
**Granularity:** Coarse

---

## Phase 1: 数据模型与文件上传接口

**Goal:** 扩展现有 rehearsal 数据模型以支持文件上传来源，建立上传 API 和前端入口，完成文件校验与持久化。

**Requirements:**
- MODEL-01: 围绕 rehearsal_session + rehearsal_scene 建模，增加会话级上传来源信息
- MODEL-02: 每个页级场景保留完整元数据（原始页码、跳过状态、页图、讲稿、音频等）
- MODEL-03: 播放页总可播放页数基于可播放场景数
- MODEL-04: 会话详情可看到原始总页数、跳过页数、失败兜底页数等摘要
- UPLOAD-01: 用户可在预演模块内上传 PDF/PPT/PPTX 文件发起预演
- UPLOAD-02: 上传文件大小上限 50MB
- UPLOAD-03: 上传文件页数上限 30 页
- UPLOAD-04: 上传入口独立于预演模块
- UPLOAD-05: 用户上传后不再补充文本输入
- STATE-01: 页面状态区分可播放页、跳过页、兜底页、不可恢复页

**Success Criteria:**
1. 数据库迁移成功运行，rehearsal_session 包含上传来源字段，rehearsal_scene 包含页级元数据字段
2. 前端预演模块可见独立的文件上传入口，支持选择 PDF/PPT/PPTX 文件
3. 上传接口对 >50MB 文件和 >30 页文件分别返回明确错误提示
4. 上传成功后文件持久化到 OSS，会话记录写入数据库

---

## Phase 2: 文件处理与页级解析管线

**Goal:** 实现 PPT→PDF 转换、PDF 页级渲染为图片、页级文本提取、空白页/封底页自动检测跳过的完整文件处理管线。

**Requirements:**
- PROC-01: PDF 文件直接进入页级处理链路
- PROC-02: PPT/PPTX 先通过 LibreOffice 转 PDF
- PROC-03: PPT→PDF 转换失败时整会话失败，错误提示明确
- PROC-04: PDF 每页通过 PyMuPDF 渲染为图片，上传 OSS
- PROC-05: PDF 每页提取文本内容
- STATE-02: 自动跳过空白页和封底页，保留跳过记录和原因
- STATE-03: 跳过页不进入播放序列，但在会话中可见
- STATE-04: 页图成功但后续失败时保住可播放性
- STATE-05: 部分页失败整会话仍可播放
- HIST-03: 原始文件、PDF、页图、讲稿、音频作为会话资产保存
- HIST-04: 会话资产不因页面刷新或会话关闭而丢失

**Success Criteria:**
1. 上传 PPT/PPTX 文件后成功转换为 PDF，转换失败时返回明确的"转换失败"错误
2. PDF 每页成功渲染为图片并上传到 OSS，空白页和封底页被标记为跳过并记录原因
3. 每页文本内容成功提取，可用于后续讲稿生成
4. 部分页处理失败时，已成功的页仍保持可用状态，整会话不因个别页失败而中断

---

## Phase 3: 讲稿生成与 TTS 语音合成

**Goal:** 基于页级解析结果调用 LLM 生成教学化讲稿，通过现有 TTS 链路合成语音，实现失败降级和 SSE 流式推送。

**Requirements:**
- SCRIPT-01: 为每个可播放页生成教学化讲稿，结合页级解析和整体上下文
- SCRIPT-02: 讲稿风格像老师现场讲解，不脱离原页内容
- SCRIPT-03: 为每个讲稿生成 TTS 语音，沿用现有 DashScope TTS
- SCRIPT-04: TTS 失败时页面仍可通过文本讲稿播放
- SCRIPT-05: 讲稿主生成失败时退化生成简化讲稿
- PLAY-06: 后端用 SSE 长连接流式推送页级进度

**Success Criteria:**
1. 每个可播放页生成的讲稿内容紧密围绕原页内容，风格为教师口语化讲解
2. TTS 语音成功生成并上传 OSS，与对应场景关联
3. LLM 讲稿生成失败时自动退化为简化讲稿，TTS 失败时页面降级为文本播放模式
4. 后端通过 SSE 实时推送每页的处理进度（解析/讲稿/语音），前端可实时展示

---

## Phase 4: 前端播放适配与历史集成

**Goal:** 播放页支持图片模式渲染上传预演，加载页适配文件处理进度文案，历史页区分来源并支持回放。

**Requirements:**
- PLAY-01: 用户确认后进入现有预演加载页
- PLAY-02: 加载页文案体现"文件解析/页级讲稿/语音生成"
- PLAY-03: 生成完成后进入现有播放页
- PLAY-04: 播放页对上传预演切换为图片模式
- PLAY-05: 翻页由系统自动控制，动作收敛为 speech + 翻页
- HIST-01: 上传预演与普通预演在历史页并存，体现来源
- HIST-02: 历史页可再次打开会话并继续播放

**Success Criteria:**
1. 上传文件后点击确认进入加载页，加载页显示文件处理相关进度文案
2. 播放页对上传预演直接展示原页图片而非 elements 渲染，翻页和语音正常工作
3. 历史页列表中上传预演显示"文件上传"来源标识，可点击进入播放
4. 从历史页打开已生成的上传预演会话，所有资产完整可播放
