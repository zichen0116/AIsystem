# EDUPrep 首页迁移设计

## 概述

本设计将 `teacher-platform` 的首页替换为 `D:\123\EDUPrep\EDUPrep` 首页的完整视觉与内容版本，并将其迁移到当前项目的 `Vue 3 + vue-router + Pinia` 架构中。

迁移目标不是简单模仿部分样式，而是完整迁移以下内容：

1. 首页整体视觉语言，包括深色主题、背景网格、发光层、卡片质感和排版节奏。
2. 首屏滚动开幕效果，包括左右幕布展开、标题显现、按钮进入和沉浸式首屏布局。
3. 中后段内容结构，包括愿景说明、核心能力矩阵、closing slogan、页脚团队署名和版权文案。
4. 首页文案与品牌表达，包括 `EDU Prep` 相关标题、说明文字、按钮文本和 footer 文本。

迁移后的页面仍然属于当前站点，因此所有交互行为需要接回现有站内逻辑，而不是直接沿用目标项目中的空壳交互。

## 设计目标

1. 在当前项目中还原 `EDUPrep` 首页的核心视觉体验和内容结构。
2. 保持当前项目既有登录、用户态、路由跳转逻辑不变。
3. 不引入额外动画依赖，使用 Vue 原生能力和现有 CSS 实现滚动动画。
4. 保证首页在桌面端具有高还原度，在移动端可稳定退化。
5. 将改动尽量限制在首页相关文件，避免对其他业务页造成回归风险。

## 范围

### In Scope

- 重写 [teacher-platform/src/views/Home.vue](d:/123/AIsystem/teacher-platform/src/views/Home.vue) 的首页模板、脚本和样式。
- 迁移目标首页的一页式内容结构。
- 迁移首页文案，包括首屏标题、说明、功能区标题、closing slogan、页脚团队署名和版权信息。
- 将主 CTA、登录入口、头像入口接回当前项目已有逻辑。
- 用 Vue 滚动监听和 CSS `transform/opacity/filter` 实现首屏开幕动画。
- 为桌面端和移动端补齐响应式布局。

### Out of Scope

- 不迁移目标项目的 React、Tailwind、`motion/react` 技术栈。
- 不将当前项目改造成目标项目的多页面或整体主题系统。
- 不修改非首页业务页的视觉设计。
- 不新增首页以外的后端接口或数据结构。

## 现状与约束

### 当前项目

- 首页路由为 `/`，页面文件为 [teacher-platform/src/views/Home.vue](d:/123/AIsystem/teacher-platform/src/views/Home.vue)。
- 登录态依赖 `useUserStore()`，未登录时首页右上角点击登录，已登录时点击头像进入个人中心或管理员资料页。
- 首页 CTA 目前会跳转到 `/lesson-prep`，且未登录时会先跳转 `/login?redirect=/lesson-prep`。

### 目标项目

- 目标首页使用 React 组件拆分、Tailwind 原子类和 `motion/react` 的滚动驱动动画。
- 目标首页重点在沉浸式视觉与首屏滚动体验，而非复杂业务逻辑。

### 实施约束

- 当前项目未引入专门的滚动动画库，因此不应为了单页迁移额外引入新依赖。
- 当前首页需要兼容现有登录态、路由行为和站点入口定位。
- 外链图片资源存在稳定性风险，迁移时优先替换为本地可控资源或可接受的现有替代方案。

## 迁移后的信息架构

迁移后的首页保持一页式长页面结构，按以下顺序组织：

1. Topbar
2. Curtain Hero
3. Vision Section
4. Features Section
5. Closing Section
6. Footer

### 1. Topbar

- 左侧显示品牌标识与 `EDU Prep` 品牌名。
- 右侧保留当前项目的登录按钮或用户头像按钮逻辑。
- 在视觉上切换为目标站风格，包括深色底、简洁边框、轻微玻璃或半透明感。

### 2. Curtain Hero

- 首屏采用目标站的开幕式视觉结构。
- 中央内容包括：
  - 标签型前缀文案
  - 中英混排主标题
  - 英文副标题与中文标语
  - 主 CTA 按钮
- 左右两侧为“数字幕布”层，随滚动逐渐向两边展开。
- 背景包含深色底、网格纹理、轻微发光球层。

### 3. Vision Section

- 保留目标站“范式转变”叙事结构。
- 左侧为说明文案和能力标签，右侧为视觉展示区。
- 如果目标项目使用的外链图片不稳定，则替换为当前项目可控的本地插画或已有资源，同时保留同等版式与氛围。

### 4. Features Section

- 迁移目标站的四张能力卡片。
- 每张卡片包含英文小标题、中文标题和说明文本。
- 视觉采用深色毛玻璃卡片、边框和局部高光。

### 5. Closing Section

- 居中展示收束性的品牌口号与英文副文案。
- 延续深色沉浸氛围，作为页面情绪上的收尾。

### 6. Footer

- 完整迁移页脚品牌展示、团队署名和版权文案。
- 若目标文案与当前项目品牌存在冲突，以用户要求为准，按“完整迁移”处理。

## 交互设计

### 登录与用户态

- 未登录时：
  - 右上角按钮显示登录入口。
  - 点击后继续调用当前首页已有的登录弹窗或登录跳转逻辑。
- 已登录时：
  - 右上角显示用户头像首字母。
  - 点击后继续按照当前逻辑进入个人中心或管理员资料页。

### 主 CTA

- 首屏主按钮文案采用目标首页版本。
- 点击行为仍然映射到当前项目的备课入口，即复用 `goLessonPrep()` 或等效逻辑。

### 页面滚动

- 首屏动画由滚动进度驱动。
- 当用户在首页顶部向下滚动时：
  - 左右幕布分别向两侧移动。
  - 幕布宽度和倾斜角度逐渐变化。
  - 中央 hero 内容从低透明度、轻微下移和缩放状态过渡到完全显现。
- 动画只绑定首页首屏区域，不影响下方 section 的常规滚动。

## 组件与代码组织

优先采用单页内清晰分区实现，避免过度拆分带来的上下文跳转成本。

第一阶段推荐直接在 [teacher-platform/src/views/Home.vue](d:/123/AIsystem/teacher-platform/src/views/Home.vue) 中完成：

- `script setup`
  - 用户态和路由逻辑
  - 滚动监听
  - hero 动画状态计算
  - 首页静态内容配置
- `template`
  - 按 section 明确分块
- `style scoped`

如果文件体积明显超出可维护范围，可以在第二步拆出首页子组件，但不是本次设计的前置要求。

## 动画实现方案

采用 Vue 原生滚动监听实现，不新增动画依赖。

### 动画来源

- 使用 `window.scrollY` 或首屏容器相对视口位置计算滚动进度。
- 将进度标准化为 `0 ~ 1`。

### 动画映射

- 左幕布：
  - `translateX(0 -> -100%)`
  - `scaleX(1 -> 0.5)`
  - `skewX(3deg -> -12deg)`
- 右幕布：
  - `translateX(0 -> 100%)`
  - `scaleX(1 -> 0.5)`
  - `skewX(-3deg -> 12deg)`
- Hero 内容：
  - `opacity(0 -> 1)`
  - `translateY(50px -> 0)`
  - `scale(0.9 -> 1)`

### 动画边界

- 首屏区域提供足够滚动空间，用于承载幕布展开效果。
- 超出动画段后，下方 section 正常滚动，不再继续增加动画复杂度。

### 性能要求

- 滚动监听只更新必要的响应式状态。
- 尽量使用 `transform` 和 `opacity`，避免触发布局抖动。
- 在移动端减少过度 blur 和大型阴影，避免掉帧。

## 视觉设计约束

### 颜色

- 页面主背景切换为接近目标站的深色。
- 强调色使用亮青色系。
- 文本分层保留高对比主文字和低对比次级文字。

### 材质

- 使用深色玻璃卡片、柔和描边和发光背景层。
- 保留目标站的网格背景和数字幕布纹理。

### 字体与排版

- 当前项目未引入目标项目同级字体系统时，使用现有系统字体做尽量接近的权重与字距处理。
- 保证大标题、英文小标签和正文层次明确。

## 响应式策略

### 桌面端

- 优先保证与目标站接近的视觉比例。
- Hero 保持左右对称与居中聚焦。

### 平板端

- 缩小标题和幕布尺寸。
- 允许 section 纵向压缩但保持内容顺序不变。

### 手机端

- Hero 改为更紧凑的垂直结构。
- 幕布动画简化，优先保留透明度与位移效果，不强求完整透视感。
- 功能卡片改为单列排列。
- Footer 文案改为纵向堆叠。

## 资源迁移策略

### 文案

- 目标首页的标题、说明、section heading、closing slogan、团队署名和版权文案全部迁入当前首页。

### 图像

- 若目标项目使用外链图片：
  - 优先评估是否可直接替换为本地资源。
  - 如果当前仓库已有可用插画或动画资源，优先使用本地资源保证稳定性。
  - 如果没有同类资源，则保留版式结构并采用可控替代视觉。

### 图标

- 目标项目使用的图标若来自 React 图标库，需在 Vue 中改为文本符号、CSS 图形、现有资源或内联 SVG 实现。

## 风险与缓解

### 风险 1：React 动画无法一比一复刻

- 缓解：保留视觉意图和关键过渡，使用 Vue 原生滚动动画做高相似度实现，而不是强求技术实现一致。

### 风险 2：外链图片不可控

- 缓解：优先改为本地资源或可控替代素材。

### 风险 3：首页单文件过大

- 缓解：先按 section 明确组织；若实现后体积过大，再拆分为首页子组件。

### 风险 4：深色视觉与现有站点风格差异较大

- 缓解：将改动严格限定在首页，不扩散到其他业务页面。

## 实施结论

推荐方案为：

`在现有 Vue 首页中，完整迁移 EDUPrep 首页的内容与视觉表现，并将所有交互接回当前项目逻辑，不新增动画依赖。`

该方案兼顾了目标还原度、工程稳定性和后续维护成本，是当前项目下风险最低、可交付性最高的实现路径。
