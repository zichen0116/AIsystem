# 教学意图确认工作流设计方案

## 1. 背景与目标

现有方案（方案1）：用户在首页输入主题和选择模板后，进入对话页与 AI 讨论教学意图，讨论几轮后 AI 直接生成大纲，用户在大纲页修改。

**问题**：意图没经过用户确认，AI 理解偏差无法及时发现，带着偏差生成大纲后修改代价大。

**改进方案（方案2）**：对话页 = 意图确认页 → 用户确认意图 → 带着明确意图进大纲页生成大纲。

## 2. 流程对比

```
方案1（现有）:
  Home → Dialog(讨论意图) → 生成大纲 → Outline → Description → Preview

方案2（改进后）:
  Home → Dialog(讨论意图) → 用户确认意图 → Outline(生成大纲) → Description → Preview
```

**关键区别**：大纲生成从 Dialog 页移到 Outline 页，Dialog 页只负责意图确认。

## 3. 对话策略：如何提问

AI 每轮主动抛出一组问题（2-3个），避免逐个问效率低。

### 第一轮：建立基础（2-3个问题）

> **AI**：
> 太好了，收到你的主题了！为了把这个 PPT 做得真正适合你的课堂，我来和你聊几个小问题，不用紧张，我们慢慢来 :)
>
> 1. **学生情况**：方便说说你的学生是哪个年级/什么专业吗？他们对这个主题大概了解多少？
> 2. **课时安排**：这节课大概多长时间呀？页数上有没有什么想法？
> 3. **教学风格**：你喜欢偏严谨一点的风格，还是轻松活泼一些的呢？

### 第二轮：明确目标（2-3个问题）

> **AI**：
> 很好，这些信息很有帮助！接下来我们聊聊教学的核心 :)
>
> 1. **学习目标**：你希望通过这节课，学生具体能掌握什么？能举个例子说说吗？
> 2. **重点难点**：你觉得这节课里，哪些地方学生会觉得比较难理解？
> 3. **先备知识**：学生需要提前知道些什么吗？要不要先帮他们复习一下？

### 第三轮：细化要求（2-3个问题）

> **AI**：
> 太好了，框架已经慢慢清晰起来了！最后再确认一些小细节就好啦~
>
> 1. **互动设计**：课堂上有没有想设计一些提问、讨论或者小测环节呀？
> 2. **内容限制**：有没有什么是你特别想加进去，或者特别想避免的内容呢？
> 3. **素材需求**：需不需要配一些图片或案例呀？有没有现成的资料想放进去？

### 第N轮：综合整理

当 AI 判断信息足够时，主动整理结构化意图，语气温暖友好：

> **AI**：
> 太好了！经过我们的讨论，我已经把你的想法整理出来啦，请看看有没有需要调整的地方：
>
> ---
>
> **主题**：《人工智能：从规则系统到大模型》
>
> **受众**：高一信息技术课的同学，他们大概知道 AI 是什么，但对发展脉络还不太清楚
>
> **学习目标**：
> - 能说出 AI 发展的三个阶段
> - 能举例大模型在生活中的应用
>
> **课时**：45分钟，大概10页左右
>
> **风格**：语言尽量通俗易懂，避免复杂公式，结尾可以加一个课堂小测
>
> **互动**：计划2次提问 + 1个生活案例讨论
>
> ---
>
> 如果看起来没问题，点下面的「确认意图，进入大纲页」就可以啦。
> 如果还想调整，随时告诉我，我们继续聊 :)



## 4. AI 对话语气

**核心原则**：温柔、有耐心、不催促。像一位耐心的教学顾问，引导而非审问。

**具体要求**：
- 不用"必须"、"应该"等命令式语气，多用"我们来..."、"你觉得..."、"方便的话..."
- 不一次性抛出一堆问题让用户压力山大，而是2-3个一组，逐步引导
- 用户回答后要先肯定再补充，比如"很好，这对设计很有帮助，同时..."
- 当用户犹豫或不知道时，要给台阶下，比如"没关系，我们可以先跳过这个，后面再补充"
- 每轮结束给正向反馈，让用户感觉进展顺利

## 5. AI 问题引导维度

| 维度 | 示例问题 |
|------|---------|
| **受众** | 学生是几年级？什么专业？零基础还是有基础？ |
| **目标** | 学完这节课学生应该掌握什么？会做什么？ |
| **课时** | 多少课时？每课时多长？打算多少页？ |
| **风格** | 偏正式严谨，还是轻松活泼？要不要游戏互动？ |
| **重点难点** | 哪些知识点要重点讲？学生容易在哪犯错？ |
| **先备知识** | 学生需要什么基础？要不要先复习？ |
| **互动设计** | 要不要提问、讨论、小测？大概几次？ |
| **内容限制** | 有没有必须包含或必须避免的内容？ |
| **素材** | 要不要配素材？有没有现成资料要插入？ |
| **载体** | 是录课、视频课、还是直播课？ |

## 5. 前端改动

### Dialog 页改动

**标题**：`教学意图确认工作台`

**流程条**：
- Step 1：模板与主题已锁定 ✓
- Step 2：教学意图反复澄清（当前）
- Step 3：生成大纲并切分页面
- Step 4：大纲页/描述页/预览页

**右侧意图看板**：
- 实时显示 AI 提取的意图要素
- 随着对话深入逐步完善
- 最后一轮显示完整结构化意图摘要

**底部 Actions**：

| 按钮 | 文字 | 状态 |
|------|------|------|
| 主按钮 | 「确认意图，进入大纲页」 | 始终可点（对话至少1轮后） |
| 次按钮 | 「继续讨论」 | 始终可点，用户可继续补充 |

**注意**：不再有「生成初版大纲并展示」按钮，大纲生成移到 Outline 页。

### 数据流

1. Dialog 页：`state.confirmed` 中的意图要素作为结构化数据存入 `pptStore`
2. 跳转 Outline 页时：把确认好的意图作为 context 传递给大纲生成
3. Outline 页：基于确认的意图生成大纲（不在 Dialog 页生成）

## 6. 后端改动

### 对话 API (`POST /chat`)

响应中 `intent_state` 保持现有结构，新增一个 `intent_summary` 字段，当 AI 判断信息足够时返回完整结构化摘要：

```json
{
  "message": "经过几轮讨论，我整理出以下教学意图...",
  "round": 4,
  "intent_state": { ... },
  "intent_summary": {
    "topic": "人工智能发展历程",
    "audience": "高一学生，有基础概念",
    "goal": "掌握三阶段演进，能举例应用",
    "duration": "45分钟/10页",
    "style": "通俗，避免公式",
    "interaction": "2次提问+1个案例讨论",
    "constraints": "不超过10页"
  },
  "ready_for_confirmation": true
}
```

### 新增 API（可选）

`POST /projects/{id}/intent/confirm` - 用户确认意图时调用，将意图写入项目记录。

## 7. 页面路由

```
home → dialog → outline → description → preview
```

- home: 模板/风格选择
- **dialog: 意图讨论 + 确认（核心改动）**
- outline: 基于确认意图生成大纲
- description: 生成页面描述
- preview: PPT 预览

## 8. 讨论原则

1. **AI 主导节奏**：AI 每轮主动抛出 2-3 个关联问题，引导用户补充
2. **用户可随时打断**：用户可以直接说"我觉得应该改成..."，AI 立即响应
3. **信息逐步完善**：意图看板实时更新，用户能看到自己填的内容
4. **确认后才推进**：用户点确认才进大纲页，保证意图质量
5. **容许反复**：用户确认后仍可退回来改意图重新生成大纲
